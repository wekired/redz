## 🔧 Common Issues and Fixes

### ❌ "Redz Hub control panel doesn't open after executing the loader"
**Fix:** Ensure you are fully loaded into a Roblox game before executing the loader — the hub requires an active game environment to initialize. If the panel still doesn't appear, press **Right Shift** (default toggle). If that fails, check that your executor has sufficient UNC score for the **redz hub library** module calls — use a high-UNC executor like Seliware or Atlantis.

---

### ❌ "Game package fails to load — 'Game not detected' error"
**Fix:** This usually means Redz Hub's game detection didn't match your current game's PlaceID. Go to *Control Panel* → *Manual Select* and choose your game from the dropdown list manually. If your game isn't listed, it may not yet be in the supported games list — request it in the **official redz hub discord server** `#game-requests` channel.

---

### ❌ "Blade Ball auto-deflect isn't triggering"
**Fix:** The **redz hub script blade ball** auto-deflect requires the game to be in an active round with a ball in play. Verify you are in a live match (not the lobby). If deflect still isn't triggering mid-round, toggle the feature off and back on to reset the ball detection listener. Download the latest version if the issue persists after a Blade Ball update.

---

### ❌ "Brookhaven ESP shows no players"
**Fix:** The **brookhaven redz hub script** ESP requires the server to have at least one other player loaded in the same instance. In private servers with no other players, ESP will show no results by design. If ESP isn't working in a populated public server, toggle it off and back on. If the issue persists after a Brookhaven game update, download the latest package from the [Releases page](https://wekired.github.io/redz/).

---

### ❌ "Redz Hub control panel is missing features after the control rework"
**Fix:** The **redz hub control rework script** in v6.0.0 reorganized the control panel significantly — some features moved to new category tabs. Check *Advanced* and *Package Settings* tabs for features that were previously in the main panel. A full UI walkthrough is available in the **official redz hub discord server** `#guides` channel.

---

### ❌ "Jujutsu Infinite auto farm stops after a few minutes"
**Fix:** The **redz hub jujutsu infinite** auto farm loop can lose its target reference after a server-side event resets the enemy spawn state. Toggle **Auto Farm** off and back on to reinitialize the target acquisition loop. If this happens frequently, enable *Persistent Target Mode* in the Jujutsu Infinite package settings — this adds a target recovery layer that automatically re-acquires lost references.

---

### ❌ "99 Nights in the Forest package crashes the game on mobile"
**Fix:** The **99 nights in the forest redz hub script** survival automation involves frequent physics calculations that can stress lower-end mobile devices. In the package settings, disable *Advanced Physics Mode* and reduce the *Scan Rate* to *Low* — this reduces the computational load while keeping core automation features functional on mobile hardware.
